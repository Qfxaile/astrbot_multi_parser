# astrbot_multi_parser

B站、抖音、小红书等多平台内容解析插件。

## 功能

- 自动识别 B站 BV/AV 号、短链、动态、Opus 图文和专栏。
- 自动识别大陆抖音视频、普通图文、Slides 和短链。
- 自动识别小红书视频、图文、短链和部分 JSON 分享卡片。
- 抖音和小红书直接读取平台分享页或平台接口，不依赖第三方解析服务。
- B站图文按照正文、图片和图片失败提示的原始顺序发送。
- 先发送标题、作者、简介等元数据；图片按图片槽位阈值和平台能力发送。
- 解析成功后可给原消息添加表情回应。
- B站、抖音和小红书的封面与正文图片均在对应平台解析会话内下载，并将源文件字节直接编码为 Base64 发送；插件不会缩放或转码图片。
- 当图片槽位不少于 3 个时，`aiocqhttp` 和 `satori` 会先发送不含图片的普通简介，再发送一个合并转发，每张图片、正文或失败提示各占一个节点；其他平台降级为普通消息链。
- 启用 `send_video_by_url` 且解析到视频直链时，通过 URL 单独发送视频，不下载到本地。
- 启用 URL 直发时先检查远程文件大小；超过 `max_video_size_mb`，或大小未知且 `allow_unknown_video_size` 未启用时，改用合并转发发送解析链接。

## 配置项

- `enabled_platforms`：启用的平台解析器，可选 `bilibili`、`douyin`、`redbook`。
- `douyin_cookies`：可选抖音 Cookie；缺少 `ttwid` 时插件会尝试注册匿名会话。
- `redbook_cookies`：可选小红书 Cookie，可提高部分内容和无水印资源的可用性。
- `request_timeout_seconds`：接口请求超时时间。
- `send_video_by_url`：解析到视频直链后是否直接通过 URL 发送视频。
- `max_video_size_mb`：直接发送视频的最大体积，超过后改用合并转发发送解析链接；小于等于 `0` 表示不限制。
- `allow_unknown_video_size`：无法通过 `HEAD` / `Range` 获取视频大小时，是否仍然直接发送视频。
- `size_check_timeout_seconds`：视频大小检查请求超时时间。
- `enable_parse_reaction`：解析成功后是否给原消息添加表情回应。
- `reaction_action`：OneBot 表情回应动作名，不同协议端可能不同，默认 `set_msg_emoji_like`。
- `reaction_emoji_id`：解析成功时使用的表情 ID，默认 `124`。

## 依赖

- `httpx`

## 架构

插件入口在 `main.py`，平台解析器放在 `platforms/`：

```text
platforms/
├── bilibili.py
├── douyin.py
└── redbook.py
```

后续新增平台时，新增一个解析器类并注册到 `main.py` 即可。

## 注意事项

- 插件不下载视频，不创建视频缓存。
- B站、抖音和小红书图片只会临时下载到内存，不写入本地文件或缓存；JPEG、WebP 等源文件会保持原始字节和格式。协议端接收后是否再次压缩不受本插件控制。
- 单张图片下载失败时，插件会在原位置发送“第 N 张图片获取失败”提示，不会回退使用压缩图或缩略图。
- 平台 Cookie 仅按对应平台域设置，跨域图片 CDN 请求不会携带这些 Cookie；图片请求使用必要的 Referer，错误日志只记录主机名和错误摘要，避免暴露带令牌的完整图片 URL。
- 所有平台视频仍使用远程 URL 构造消息组件。
- 插件只通过 `HEAD` / `Range` 探测视频大小，不会把完整视频下载到本地。
- 视频 URL 能否直接发送，取决于协议端和目标平台对远程视频 URL 的支持。
- 协议端拉取远程媒体时无法附带插件请求 Cookie 或 Referer，个别 CDN 直链可能失效。
- 当视频超过大小限制，或大小未知且不允许发送时，插件会使用 OneBot 合并转发发送解析链接。
- 表情回应依赖协议端支持 `reaction_action` 对应动作；失败时只记录日志，不影响解析结果发送。
- 插件会先发送解析信息，再单独发送视频；即使视频发送失败，标题和封面也能保留。
- 插件不支持 TikTok，也不解析 B站音频、独立音轨或 `au` 号。
- 平台页面结构和公开接口可能发生变化，失效时需要同步更新对应平台解析器。
